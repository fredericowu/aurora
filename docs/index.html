Architecture

Components

  • ▸ API Gateway: HTTP API exposing search endpoints
  • ▸ Lambda: FastAPI application with Mangum adapter
  • ▸ RDS PostgreSQL: Messages stored with tsvector for search
  • ▸ VPC: Private network with security groups
  • ▸ GitHub Actions: Automated deployment and ingestion

Database Schema

CREATE TABLE messages (
    id VARCHAR PRIMARY KEY,
    user_id VARCHAR NOT NULL,
    user_name VARCHAR NOT NULL,
    timestamp TIMESTAMP NOT NULL,
    message TEXT NOT NULL,
    search_vector TSVECTOR
);

CREATE INDEX idx_search_vector
ON messages USING GIN(search_vector);

Quick Start

1. Configure Environment

Create a .env file with your AWS credentials:

cp .env.example .env
# Edit .env with your values:
# - AWS_ACCESS_KEY_ID
# - AWS_SECRET_ACCESS_KEY
# - AWS_REGION
# - DB_PASSWORD

2. Deploy Infrastructure

Deploy all AWS resources with a single command:

make deploy

This takes ~10-15 minutes and creates: VPC, RDS, Lambda, API Gateway

3. Setup Environment Variables

Populate .env from Terraform outputs:

make setup-env

4. Ingest Messages

Load messages from the external API:

make ingest

5. Run Performance Tests

Validate response times and performance:

make performance-test

Performance

Optimization Techniques

  • GIN Index: Generalized Inverted Index on tsvector column
  • Connection Pooling: Reuse database connections in Lambda
  • Prepared Statements: Query plan caching for efficiency
  • Efficient Pagination: Limit result sets to reduce transfer time
